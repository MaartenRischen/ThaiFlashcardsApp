Donkey Bridge: Set Generation Logic (Linguistic & Prompt Focus)

1. Overview: From User Input to Flashcard Set

The heart of Donkey Bridge is the transformation of user intent into a linguistically rich, ultra-personalized Thai flashcard set. This page explains how your choices become LLM prompts, how those prompts are crafted, and how the results are reflected in the app UI.
  1. User configures a set in the Set Wizard (proficiency, topic, tone, card count).
  2. The app constructs a detailed LLM prompt encoding all linguistic rules and user intent.
  3. The LLM returns a JSON array of phrases, each with fields for Thai, English, pronunciation, mnemonic, and examples.
  4. The app validates, cleans, and stores the results, then displays them in the UI.

2. User Input → Prompt Construction

2.1. User Choices

  • Proficiency Level: Affects phrase complexity, grammar, and vocabulary.
  • Topic/Goal: The main subject for all generated phrases (single string, e.g., "Ordering vegetarian food").
  • Tone (1-10): Controls creativity/chaos (mapped to LLM temperature and prompt instructions).
  • Card Count: Number of phrases to generate (5, 10, 15, or 20).

2.2. Prompt Structure & Linguistic Rules

The prompt sent to the LLM is a carefully engineered instruction set. It encodes:

  • Proficiency: "Generate phrases suitable for a [level] Thai learner."
  • Topic: "All phrases must be directly relevant to: '[topic]'."
  • Tone: "If tone is high, allow more humor, creativity, and surrealism. If low, keep practical and realistic."
  • Politeness: "Add the correct polite particle (ครับ/ค่ะ) unless the phrase is a question or already ends with one."
  • Gender: "Use gender-appropriate pronouns and endings as needed."
  • Mnemonic: "For each phrase, provide a creative mnemonic (memory aid) in English, never just a translation."
  • Examples: "For each phrase, provide at least one example sentence in Thai, with English translation and pronunciation."
  • Output Format: "Return a JSON array of objects with fields: english, thai, pronunciation, mnemonic, examples[]."
  • Title: "Suggest a clever, short set title related to the topic."
  • Constraints: "No English in the Thai field, no Thai in the English field, no empty fields, no duplicates."
The prompt is dynamically constructed from the user's choices and always includes all relevant linguistic rules.

2.3. Prompt Example (Annotated)

Generate 10 Thai flashcard phrases for a [Intermediate] learner about [Ordering vegetarian food]. - Each phrase must be directly relevant to the topic. - Use practical, real-world language. - Tone: 2 (Serious & practical) — keep phrases realistic and useful. - Add the correct polite particle (ครับ/ค่ะ) unless inappropriate. - Use gender-appropriate pronouns. - For each phrase, provide: - English translation - Thai phrase - Pronunciation (Latin script) - A creative mnemonic (memory aid) in English - At least one example sentence (Thai, English, pronunciation) - Output as a JSON array of objects with fields: english, thai, pronunciation, mnemonic, examples[] - Suggest a clever, short set title related to the topic. - No English in Thai, no Thai in English, no empty fields, no duplicates.

3. LLM Response → App Data & UI Mapping

3.1. Expected LLM Output

[ { "english": "Can I have the vegetarian menu?", "thai": "ขอเมนูมังสวิรัติได้ไหมครับ", "pronunciation": "kho menu mang-sa-wi-rat dai mai krap", "mnemonic": "Imagine a menu with only vegetables and a monk (mang) serving it.", "examples": [ { "thai": "ขอเมนูมังสวิรัติด้วยครับ", "english": "Please give me the vegetarian menu, too.", "pronunciation": "kho menu mang-sa-wi-rat duay krap" } ] }, ... ]

3.2. Validation & Cleaning

  • Each phrase is checked for required fields, type, and non-empty values.
  • Politeness and gender are checked and fixed if needed.
  • Mnemonics are checked for creativity (not just translation).
  • Examples are checked for completeness and relevance.
  • Fallbacks: If the LLM output is incomplete, fallback phrases are added from a static list.

4. Image Generation Logic

The set cover image is generated by Ideogram using a prompt based on the set title/topic. Tone does not affect the image.
  • Prompt: "Cartoon style illustration featuring a friendly donkey and a bridge, related to the topic: '[cleverTitle or topic]'. Use vibrant colors. Absolutely no text, letters, or numbers."
  • Strict anti-text/anti-letters/anti-numbers rule is always included.
  • If image generation fails, a fallback prompt or static image is used.

5. Code & Data Flow (for Developers)

  1. User completes Set Wizard → POST /api/generate-set
  2. API validates input, calls generateCustomSet (set-generator.ts)
  3. Prompt is constructed and sent to LLM (OpenRouter, Gemini, GPT-4, etc.)
  4. LLM response is parsed, validated, and cleaned
  5. Image prompt is constructed and sent to Ideogram
  6. Set metadata, phrases, and progress are saved to the database (storage.ts, Prisma/Supabase)
  7. UI fetches and displays the new set, mapping all fields as above

6. Error Handling & Fallbacks

  • If LLM output is incomplete or invalid, fallback phrases are added from a static phrase list.
  • If image generation fails, a fallback prompt or static image is used.
  • All errors are logged and surfaced to the user as needed.
